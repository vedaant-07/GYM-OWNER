# SE7EN FIT Gym Owner Website

Production gym-owner management website for SE7EN FIT.

## Current production backend

This website is now connected to the SE7EN FIT Render backend:

```bash
https://se7enfit-original.onrender.com
```

The backend uses Render + TiDB/MySQL. The frontend stores the JWT token in localStorage and sends it as a Bearer token for authenticated API calls.

## Local setup

```bash
npm install
cp .env.example .env.local
npm run dev
```

`.env.local`:

```bash
VITE_API_BASE_URL=https://se7enfit-original.onrender.com
```

## Production build

```bash
npm run build
npm run preview
```

## What was changed from Base44 mode

The original project was generated by Base44. The production version now includes a Render API adapter so the same UI can run against the live backend instead of Base44 entities.

Important files:

```text
src/lib/api-client.js          # Fetch client, JWT storage, JSON/HTML response safety
src/api/base44Client.js        # Compatibility adapter mapped to Render endpoints
src/lib/AuthContext.jsx        # JWT auth context for owner login/register/session restore
src/pages/Login.jsx            # Owner email/password login connected to backend
src/pages/Register.jsx         # Owner registration connected to backend
src/index.css                  # Neon green SE7EN FIT theme
```

## Notes

- Email/password owner login is connected to `/api/auth/login` with role `gym_owner`.
- Owner registration is connected to `/api/auth/register` with role `gym_owner`.
- Google login on the website is disabled until web OAuth is configured separately.
- Pages still using entity lists are routed through the adapter. If a backend route is not built yet, the page receives an empty list instead of crashing.
- Yellow accent colors have been replaced with neon green.
